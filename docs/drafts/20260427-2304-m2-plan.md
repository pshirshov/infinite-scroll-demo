### PR-14 — Selection bug: DOM-order fix + Playwright regression

**Root-cause hypothesis (high confidence).** `src/components/ChatViewport.tsx`
populates `rowsToRender` as `[topIndex..topIndex+below_overscan,
topIndex-1..topIndex-overscanAbove]`. DOM children follow this order, but
visual order (via `transform: translateY`) is `[topIndex-overscanAbove..
topIndex+overscan_below]`. Browser drag-selection walks DOM order →
unrelated above-rows get included.

**Fix.** Add `rowsToRender.sort((a, b) => a.topPx - b.topPx)` before the
JSX `.map`. Two lines, deterministic, makes DOM order match visual order
regardless of how loops are arranged. Document as new invariant **I-7**:
DOM children of `.chat-viewport__rows` are ordered by `topPx` ascending.

**Files.**
- `src/components/ChatViewport.tsx` — sort + invariant comment.
- `e2e/selection.spec.ts` (new).

**Sub-tasks.**
- D-14.1 Add the sort + I-7 comment in the file's invariants header.
- D-14.2 Author `e2e/selection.spec.ts`:
  (a) `goto('/')`; wait for ≥5 `.chat-message` with non-empty bodies.
  (b) Pick the 3rd visible message by `data-index`. Read its body text.
  (c) Programmatic probe via `page.evaluate`: build a `Range` covering only that body, install via `selection.addRange`, assert `selection.toString()` equals the body.
  (d) Mouse-drag probe: `page.mouse.move/down/move/up` within the row's bbox; assert resulting selection is contained in the row's body.
- D-14.3 Confirm test FAILS before fix.
- D-14.4 Apply the fix; confirm test passes.

**Acceptance.** `e2e/selection.spec.ts` passes; sticky-date headers still work; 201 unit tests still pass.

### PR-15 — Empty-bodies after scrollbar drag: reproduce, diagnose, fix

**Goal.** Reproduction-first. Do NOT pre-commit to a fix; diagnose from the failing repro.

**Files (candidate set; final list depends on D-15.2 outcome).**
- `e2e/jump-then-render.spec.ts` (new).
- Most likely: `src/components/ChatViewport.tsx`, `src/store/fetchCoordinator.ts`, `src/store/ChatStore.ts`.

**Sub-tasks.**
- D-15.1 Author the failing repro:
  - `goto('/')`; wait for live-tail anchor.
  - Locate scrollbar thumb + track (read `src/components/CustomScrollbar.tsx` for class names).
  - `page.mouse` drag thumb from bottom to ~50% of track.
  - Poll `.chat-skeleton` count; expect to settle to 0 within 5 s.
  - Assert all visible `.chat-message__body` texts are non-empty.

- D-15.2 Diagnose. Candidate probes in priority order:
  1. **Debounce reset during drag.** Instrument `scheduleEnsureRange`'s timer fire with `console.log` (DEV-only). Drag in test, inspect logs.
  2. **`abortFetchesOutside` off-by-one.** Instrument it to log keep-window vs aborted-chunk ranges.
  3. **`onChunk` race-guard over-suppression.** Instrument FetchCoordinator's `.then` disposed/aborted gate.
  4. **Region integrity violation.** Assert `messages[i].index === startIndex + i` at runtime when an empty-body row is rendered.
  5. **MessageRow rendering with empty body.** Assert non-empty at render time.
  6. **Skeleton zero-height race.** Verify `getHeight(i)` returns estimate for unloaded indices.

- D-15.3 Apply minimal fix. Record root cause in `defects.md`.
- D-15.4 Re-run all tests.
- D-15.5 If diagnosis reveals an invariant violation, codify it in the file's invariants comment + add a unit test at the store/coordinator level.

**Acceptance.** Repro test passes after fix; manual repeat confirms; unit tests still ≥201.

**Verify.** `pnpm e2e -- --grep jump-then-render`, then full suite.

**Deps.** PR-13.

## 5. Risks & assumptions

- **R1: Playwright version mismatch with nix driver.** Mitigate by aligning npm to nix; fallback `nix flake update`. Hard fallback: drop nix-managed browsers, document `npx playwright install` as a one-time step (sacrifices AC-M2-1).
- **R2: First nix eval is slow** (Chromium download, hundreds of MB). Document; warm cache.
- **R3: Vitest accidentally picks up e2e specs.** Mitigated by `*.spec.ts` vs `*.test.ts` naming; verify.
- **R4: Bug A root cause not in easy-to-instrument layer.** Escalate via `git bisect` against M1 commits.
- **R5: PR-14's sort changes React reconciliation.** Keys are `index` (stable); React reconciles on key not on order. No remount.
- **R6: `transform: translateY` interacts with Selection.** If sort doesn't fully fix Bug B, add `user-select: text` to `.chat-message` and `user-select: none` to overlays. Defer until proven needed.
- **A1: `NewMessageEvent` 5s tick is harmless to e2e specs** (each completes <5 s).
- **A2: `.chat-scrollbar__thumb` is a stable selector.** Executor confirms by reading the file.
